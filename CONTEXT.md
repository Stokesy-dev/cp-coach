# Domain Glossary: CPCoach

### Problem ID
A unique identifier for a competitive programming problem. For Codeforces problems, this is represented by concatenating the contest ID and the problem index (e.g., `1234D`).

### Normalized Submission
A standardized record representing a user's attempt at a competitive programming problem, containing the Problem ID, a list of associated tags, the problem rating, the attempt's verdict (solved or failed), and the timestamp of the attempt.

### Topic Tag
A classification label assigned to a competitive programming problem indicating the mathematical or algorithmic concepts required to solve it (e.g., `dp`, `math`, `graphs`, `greedy`).

### Weakness Score
A numerical value computed for each Topic Tag based on the user's solve rate and total unique attempts. It ranks tags to identify which concepts the user struggles with the most, weighting higher attempt counts more heavily so that consistent failure carries more weight than isolated failure.

### Target Rating
The difficulty level (represented as a multiple of 100) currently targeted for a user in a specific Topic Tag. It represents the difficulty of problems we aim to recommend next for that topic.

### Rating Band
The range of problem ratings centered around the Target Rating that is queried when recommending a problem. If no unsolved problems match the exact Target Rating, the band is progressively widened (e.g., ±100, then ±200) until matching problems are found.

### Difficulty Adjustment
The feedback-driven loop where the Target Rating is updated based on user feedback. A "pass" (user successfully solved the problem) increases the Target Rating for that tag by 100, while a "fail" (user failed to solve the problem) decreases it by 100 (clamped to a minimum of 800).

### Recommendation History
A record of all recommended problems, their target ratings, and the user's feedback (pass or fail) submitted during the current session.

### Session State
The persistent, in-memory state of the LangGraph agent, containing the user's solved/attempted history, calculated weakness rankings, current target ratings per tag, recommendation history, and the current active recommendation.

### Feedback Loop
The conditional routing logic in the agent's graph. Upon receiving user feedback:
- A "fail" routes the agent back to the Search step to recommend an easier problem under the same tag.
- A "pass" routes the agent back to the Weak-Area Analysis step to re-evaluate the user's overall weaknesses (as their skill in the current tag has changed).

### Recommendation Rationale
A short, 1-2 sentence explanation accompanying a recommended problem. It explains why the problem was selected, referencing the user's specific weak area and the current adjusted difficulty rating. This can be dynamically generated by a Large Language Model (LLM) or fall back to a rule-based template.
